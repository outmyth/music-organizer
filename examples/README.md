# Examples

> 🌐 **English** · [中文](README.zh.md)

This directory shows what the script's input and output structure looks like, so you can browse the GitHub repo and see the layout without having to run anything.

**All audio files (`.flac`, `.dsf`, `folder.jpg`) are 0-byte placeholders** — they exist purely to illustrate the directory tree. The `.m3u`, `.cue`, and `.json` files contain real, representative content showing the exact format the script produces.

> ⚠️ The top-level `in/` and `out/` directories in the repo root are placeholders for **your real data** (gitignored). When you run the script locally, you put your music in `in/` and the script writes to `out/`. This `examples/` directory is just for documentation.

## `examples/in/` — input layout

```
examples/in/
├── Jazz/Miles Davis/Kind of Blue/                ← normal album, 3 tracks
│   ├── 01 - So What.flac
│   ├── 02 - Freddie Freeloader.flac
│   ├── 03 - Blue in Green.flac
│   └── folder.jpg
├── Classical/Beethoven/Symphony No. 5/           ← CUE+image album
│   ├── Symphony No. 5.cue                        (real CUE content)
│   ├── Symphony No. 5.flac                       (single image file)
│   └── folder.jpg
├── Cantopop/陳百強/一見鍾情/                      ← Chinese-titled album
│   ├── 01 - 一見鍾情.flac
│   ├── 02 - 偏偏喜歡你.flac
│   └── folder.jpg
└── Test/Beyerdynamic T1/                         ← reserved Test/ category
    ├── 01 - Time After Time.dsf
    └── 02 - Cheek to Cheek.dsf
```

**What this demonstrates:**
- A normal album with cover art (`folder.jpg`)
- A CUE+image album that the script splits into individual tracks via ffmpeg
- A Chinese-titled (CJK) album — the script auto-classifies as `Mandopop` if no explicit `ARTIST_GENRE` rule (here `陳百強` matches the `Cantopop` rule)
- A `Test/{Category}/` subdirectory for headphone audition / format comparison tracks

## Inspecting the playlist format

Compare the same playlist written to two locations — the relative path prefix differs because each is anchored to its own directory:

**`playlists/All.m3u`** (from project root, music is one level up + into `MUSIC/`):
```
#EXTINF:545,Miles Davis - So What
../MUSIC/Jazz/Miles Davis/(1959) Kind of Blue/01 - So What.flac
```

**`MUSIC/Playlists/All.m3u`** (already inside `MUSIC/`, just up one level):
```
#EXTINF:545,Miles Davis - So What
../Jazz/Miles Davis/(1959) Kind of Blue/01 - So What.flac
```

This is **standard M3U / RFC 8216 path resolution** — each playlist is self-contained relative to its own location.

## How the demo files were generated

These are hand-crafted, not produced by running the script (the audio files are 0 bytes, so `ffprobe` would fail). They were created to match exactly what `music_organizer.py` would output, based on:

- **`in/` tree** mirrors the input shape the script expects
- **`out/MUSIC/`** structure follows the `{Genre}/{Artist}/({Year}) {Album}/` formula
- **`.m3u` files** use the exact format from `music_organizer.py` line 896-906 (header + EXTINF + relative path)
- **`music_index.json`** matches the schema generated by Step 7

To produce real output from real input on your own machine, see the [main README](../README.md).
